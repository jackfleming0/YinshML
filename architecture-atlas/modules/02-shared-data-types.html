The Shared Currency: Core Data Types

The exact objects, fields, and shapes passed across module boundaries.

Every module in YinshML speaks in a small set of shared types. A type produced in one file is consumed verbatim in another — the dataclass contract is the API. These are the objects that travel the seams.

Position, Move, and the three enums

The lowest layer. Player is a signed enum (WHITE=1, BLACK=-1) so its value doubles as a sign multiplier in differential heuristics.

CODE 
class Player(Enum):
    WHITE = 1
    BLACK = -1

class MoveType(Enum):
    PLACE_RING = "PLACE_RING"
    MOVE_RING = "MOVE_RING"
    REMOVE_MARKERS = "REMOVE_MARKERS"
    REMOVE_RING = "REMOVE_RING"

class GamePhase(Enum):
    RING_PLACEMENT = 0
    MAIN_GAME = 1
    ROW_COMPLETION = 2
    RING_REMOVAL = 3
    GAME_OVER = 4
PLAIN ENGLISH

Player (constants.py:11): two members, ints +1 / -1. Produced everywhere a side is named; consumed by heuristics as a literal sign. .opponent flips it.

MoveType (types.py:8): the four legal action kinds. Drives the policy-slot layout in the encoder — each kind owns a contiguous index range.

GamePhase (types.py:14): the five lifecycle states. Note ROW_COMPLETION=2 sits between MAIN_GAME and RING_REMOVAL. Consumed by encoder channel 5 (normalized by len(GamePhase)-1 = 4) and by phase-specific heuristic weights.

Invariant: these enums are the single source of truth — integer values are persisted and channel-normalized, so renumbering is a breaking change.

GameState: the mutable source of truth

Everything else is derived from this object. It is mutated in place (make_move) and copied for search (copy_from for pool reuse, avoiding deepcopy).

CODE 
@dataclass
class GameState:
    board: Board
    current_player: Player
    phase: GamePhase
    white_score: int
    black_score: int
    rings_placed: Dict[Player, int]
    move_history: List[Move]

    # row-completion bookkeeping (None when inactive)
    _move_maker: Optional[Player] = None
    _prev_player: Optional[Player] = None
    _last_regular_player: Optional[Player] = None
PLAIN ENGLISH

board — a Board holding piece placement; to_numpy_array() feeds channels 0–3 of the encoder.

current_player — whose turn; starts WHITE. Drives side-normalization in encode_state and recovery from the A1 sentinel in decode_state.

phase — current GamePhase; starts RING_PLACEMENT. Gates which moves are legal and which heuristic weights apply.

white_score / black_score — completed rows (rings removed). POINTS_TO_WIN=3 ends the game.

rings_placed — per-player count during placement; complete at RINGS_PER_PLAYER=5 each.

move_history — ordered List[Move]; replayed by the viz layer and Parquet reconstruction.

Invariant: the _* bookkeeping fields are always present (None when inactive) so pool reuse never inherits stale row-completion markers (game_state.py:127).

The encoded state tensor & network I/O

This is the seam where game logic becomes math. encode_state turns a GameState into a fixed (6,11,11) tensor; the network maps it to a policy vector and a scalar value.

CODE 
def encode_state(self, game_state) -> np.ndarray:
    state = np.zeros((6, 11, 11), dtype=np.float32)
    # 0: current rings   1: opponent rings
    # 2: current markers 3: opponent markers
    # 4: valid-move sources mask
    phase_value = game_state.phase.value / (len(GamePhase)-1)
    state[5] = phase_value          # ch 5 broadcast
    state[5, 0, 0] = sentinel   # A1: side-to-move
    return state
PLAIN ENGLISH

Shape (6,11,11) float32 (encoding.py:330). 6 channels over the 11×11 grid; only 85 cells are real, the rest are structural padding.

Channels 0–3 are side-normalized: index 0/2 are always the current player's rings/markers, 1/3 the opponent's — not white/black. The network never sees raw color.

Channel 4 is the valid-move source mask; channel 5 broadcasts the phase value, then overwrites off-board cell A1 with a 0/1 side-to-move sentinel.

Enhanced (15-ch): EnhancedStateEncoder (enhanced_encoding.py:55) keeps 0–3, adds row threats (4–5), partial rows (6–7), ring mobility (8), center distance (9), ring influence (10), valid dests (11), phase (12), turn number (13), score diff (14).

⚠
Shape invariant: side recovery lives in channel 5, not color

Because channels 0–3 are "current player first," decode_state cannot tell white from black by channel. It reads side-to-move from the A1 sentinel (row_idx=0, col_idx=0) on channel 5, and the phase from on-board cell F6 (row 5, col 5). A1 is off the hex board (column A only has rows 2–5), so the sentinel never collides with real state and survives D2 augmentation. In the 15-ch encoder the phase/sentinel move to channel CH_GAME_PHASE = 12 — magic-indexing channel 5 there reads opponent row-threats and silently breaks.

CODE 
def predict(self, state_tensor, move_mask=None,
            temperature=1.0) -> Tuple[Tensor, Tensor]:
    # state_tensor: (6, 11, 11)
    move_logits, value = self.network(state_tensor)
    if move_mask is not None:
        move_logits = move_logits.masked_fill(~move_mask, -inf)
    move_probabilities = F.softmax(move_logits/temperature)
    return move_probabilities, value
PLAIN ENGLISH

Input: a (6,11,11) tensor (wrapper.py:273). forward() (model.py:377) runs conv trunk → two heads.

Policy head (model.py:148): Linear(..., total_moves) → raw logits of length 7433. predict masks illegal slots to -inf, then softmaxes to probabilities.

Value head: a scalar — (batch,) in classification mode, (batch,1) in regression. Already tanh'd inside the model; do not tanh again.

Produced by the network; consumed by MCTS as priors + leaf value. load_model hard-fails on any policy-size mismatch — no silent slot filtering.

Magic constants that pin these shapes

total_moves = 7433 Policy-vector length. Computed as 85 + 7140 + 85 + 123 in StateEncoder.__init__ (utils/encoding.py:146); use StateEncoder().total_moves.
channels = 6 | 15 NUM_CHANNELS = 6 basic (encoding.py:103) / 15 enhanced (enhanced_encoding.py:83). Selected by use_enhanced_encoding.
board = 11×11 / 85 Grid is 11×11; only 85 cells are valid (VALID_POSITIONS, constants.py:63; column counts [4,7,8,9,10,9,10,9,8,7,4]).
RINGS_PER_PLAYER = 5 Rings placed before MAIN_GAME (constants.py:103).
POINTS_TO_WIN = 3 Completed rows to win (constants.py:105; blitz variant = 1).
num_value_classes = 7 Discrete outcome buckets for classification value head; default in NetworkWrapper.__init__ (network/wrapper.py:28).
⚠
Shape invariant: policy length MUST equal total_moves

Every policy vector — network output, MCTS target, stored Parquet row — is length 7433. The slot layout is contiguous and collision-free: placement [0,85), ring movement [85,7225) (src·84 + adjusted_dst), ring removal [7225,7310), marker removal [7310,7433) (123 enumerated 5-in-a-row lines). Changing _enumerate_remove_markers_lines() reshuffles slots and breaks every saved checkpoint; NetworkWrapper.load_model hard-fails on size mismatch rather than silently filtering. The pre-fix legacy size was 8390 — not load-compatible.

These types are produced and consumed at one critical seam: the GameState ⇄ Network boundary, where encode_state serializes a position and predict returns policy + value. Section 3 walks that seam end to end — how a live GameState becomes a tensor, crosses into the network, and returns as a move distribution the search layer can act on.
