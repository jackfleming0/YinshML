The State ↔ Network Seam

How a GameState becomes a tensor becomes a policy + value becomes a Move.

Three objects meet at this seam: a GameState, the StateEncoder, and the NetworkWrapper (which itself owns a YinshNetwork). The contract is purely shape- and index-driven: a (6, 11, 11) float array goes in, a flat move_probs[7433] distribution plus a scalar value come out, and the encoder's move-index table is the only thing that maps a policy slot back to a concrete Move.

Encode: GameState → (6,11,11)

encoding.py:323. The encoder is side-normalized: channels 0/2 are always the current player's rings/markers, 1/3 the opponent's. Channel 4 is the valid-moves mask written from each move's source cell; channel 5 broadcasts the phase and overrides the off-board A1 cell with a side-to-move sentinel.

CODE 
def encode_state(self, game_state: GameState) -> np.ndarray:
    state = np.zeros((6, 11, 11), dtype=np.float32)
    is_white = (game_state.current_player == Player.WHITE)
    # ch 0/1 rings, ch 2/3 markers: current-first
    state[0, r, c] = 1.0   # current rings
    # ch 4: valid-move sources
    for move in game_state.get_valid_moves():
        state[4, src_row, src_col] = 1.0
    # ch 5: phase broadcast + A1 sentinel
    state[5] = phase_value
    state[5, 0, 0] = WHITE_SENTINEL if is_white else BLACK_SENTINEL
    return state
PLAIN ENGLISH

Returns a dense numpy array, not a torch tensor. The wrapper does the from_numpy().float() conversion and device move.

Side-to-move decides which color lands in the "current" channels, so the network never sees raw white-vs-black.

Channels 0-3 are the four piece planes. Coordinates are (row-1, col-'A'), clamped to the 11x11 grid.

Channel 4 is the move-source mask: one hot cell per ring that can legally move this turn. This is the only "valid moves" signal baked into the tensor.

Channel 5 carries phase everywhere, then stamps the off-board A1 cell with a 0/1 sentinel so decode_state can recover side-to-move without flipping colors.

Predict: tensor → (move_probs, value)

wrapper.py:273. The single-position path. It runs self.network(state_tensor), masks invalid slots with -inf before softmax, scales by temperature, and returns probabilities plus the value. The value is not re-tanh'd here — the network already bounds it (model.py:139 region / classification expectation), so a second tanh would over-compress.

CODE 
def predict(self, state_tensor,
            move_mask=None, temperature=1.0):
    self.network.eval()
    with torch.no_grad():
        move_logits, value = self.network(state_tensor)
        # value already tanh'd by net - NO re-tanh
        if move_mask is not None:
            move_logits = move_logits.masked_fill(
                ~move_mask, float('-inf'))
        scaled = move_logits / temperature
        probs = F.softmax(scaled, dim=1)
        return probs, value
PLAIN ENGLISH

Args: a (6,11,11) (or batched) tensor, an optional boolean mask, and a temperature.

Eval mode + no-grad: this is pure inference, no autograd graph built.

One forward call yields raw policy logits of length 7433 and the value head output.

The commented-out torch.tanh(value) is the load-bearing absence: the value is bounded once, inside the net.

Masking sets illegal slots to -inf so softmax drives them to exactly zero probability.

Returns a normalized distribution over all 7433 slots and the scalar value.

The batched path MCTS actually calls

wrapper.py:617. predict_batch takes a list of GameStates, encodes each into one pooled batch tensor, runs a single forward pass, and returns raw logits (not softmaxed, not masked). This is deliberate: the caller — batched MCTS — applies its own masking and softmax per leaf. Temperature is folded into the logits when it is not 1.0.

CODE 
def predict_batch(self, game_states, temperature=1.0):
    batch_tensor = self._acquire_input_tensor(len(game_states))
    for i, gs in enumerate(game_states):
        arr = self.state_encoder.encode_state(gs)
        batch_tensor[i] = torch.from_numpy(arr).float()
    with torch.no_grad():
        policy_logits, values = self.network(batch_tensor)
        if temperature != 1.0:
            policy_logits = policy_logits / temperature
        return policy_logits, values  # raw, caller masks
PLAIN ENGLISH

Args: a list of GameState objects (one per MCTS leaf) and a temperature.

The batch input tensor comes from the TensorPool, sized (batch, 6, 11, 11), and is released in a finally block.

Each state is encoded and copied into its batch row — same encode_state as the singleton path.

One forward pass over the whole batch is the throughput win over per-leaf calls.

Returns (batch_size, 7433) logits and (batch_size, 1) values, both unmasked — masking and softmax are the search layer's job.

Inside the net: two heads, one trunk

model.py:377. forward runs the shared conv trunk, then splits. The policy head is Conv2d(c,64,1) → Flatten → Linear(64·11·11, 1024) → Linear(1024, total_moves) emitting raw logits (model.py:148). In classification value mode the value head's logits are softmaxed and dotted with outcome_values to yield an expectation already in [-1, 1] — that is the single tanh-equivalent bounding referenced by predict.

CODE 
def forward(self, x) -> Tuple[Tensor, Tensor]:
    x = self.conv_block(x)            # shared trunk
    for block in self.main_blocks: x = block(x)
    policy = self.policy_head(x)     # (B, 7433) logits
    value_output = self.value_head(x)
    if self.value_mode == 'classification':
        probs = F.softmax(value_output, dim=-1)
        value = (probs * self.outcome_values).sum(-1)
        self._value_logits = value_output  # for CE loss
    else:
        value = value_output.squeeze(-1)  # tanh'd scalar
    return policy, value
PLAIN ENGLISH

Args: (batch, input_channels, 11, 11) — 6 channels basic, 15 enhanced.

Trunk: an entry conv block then a stack of residual/attention blocks, shared by both heads.

Policy head ends in a bare Linear(..., total_moves) — raw logits, no activation, so masking and temperature happen downstream.

Classification value mode turns class logits into a probability-weighted expectation over the discrete outcome centers; the result is already in [-1,1].

The raw class logits are stashed on _value_logits so the trainer can run cross-entropy without changing this interface.

Returns (policy_logits, value) — the exact pair every predict path consumes.

Hard contracts at the seam

⚠
Policy length and channel count are load-bearing invariants

The policy head length must equal StateEncoder().total_moves == 7433 (85 + 7140 + 85 + 123, encoding.py:146). This number is the move-index layout: slot ↔ move meaning is fixed by it. NetworkWrapper.load_model (wrapper.py:423) hard-fails on three mismatches rather than silently filtering — input-channel count (wrapper.py:434), value-head mode/classes (wrapper.py:450), and policy-head size (wrapper.py:492). Silent filtering would leave the offending layer randomly initialized and play garbage. Known past policy sizes: 7395 (pre-rework), 8390 (collision-prone REMOVE_MARKERS), 7433 (current). Switching sizes means retrain or explicit migration — old checkpoints will not load.

The use_enhanced_encoding plumbing

A single boolean threads channel count from the wrapper constructor down to both the encoder choice and the network's input conv. load_model can also flip this flag to match a checkpoint's conv-in dimension (wrapper.py:78), so the encoder instantiated at wrapper.py:200 always agrees with the loaded weights.

NetworkWrapper(use_enhanced_encoding=..., num_channels=..., num_blocks=..., value_mode='classification', num_value_classes=7, value_head_type=...)

wrapper.py:26. The flag sets input_channels = 15 if use_enhanced_encoding else 6. When a model_path is given, channel count, block count, value mode, and head type are auto-detected from the state_dict and can override the flag.

2
Encoder selection — wrapper.py:200

self.use_enhanced_encoding (post-autodetect) picks EnhancedStateEncoder() (15-ch) or StateEncoder() (6-ch). Reading the post-detect flag, not the constructor arg, is what prevents an encoder/weights mismatch that would yield 0-move games.

3
YinshNetwork(input_channels=6 or 15, ...)

wrapper.py:161 → model.py:82. The same channel count sizes the entry Conv2d(input_channels, num_channels, 3, padding=1). Both sides of the seam read one source of truth, so the conv-in dimension and the encoder's output channels can never silently drift.

total_moves = 7433 85 ring placements + 7140 ring moves + 85 marker removals + 123 REMOVE_MARKERS lines (encoding.py:146).
input_channels = 6 | 15 Basic vs enhanced encoder; the only thing that changes the trunk's first conv.
value_mode = classification Expectation over 7 outcome classes in [-1,1]; load_model hard-fails on a mode mismatch.

Downstream, predict_batch is the hot path: batched MCTS collects many leaf GameStates, hands them to this seam in one shot, and applies its own masking and priors to the returned logits. That consumer — where evaluation meets search — is Section 4: the Search ↔ Evaluation seam.
