# Game review — BGA table 862307561 (Black wins 3–2)

A coaching review of a strong human YINSH game, used to surface strategic ideas
our heuristic/training stack does not yet represent. The game itself is
transcribed in `yinsh_ml/data/human_games/bga_862307561.py`; everything below is
reproducible via `scripts/review_human_game.py`.

> Scope note: this review intentionally stays abstract. It implements the
> *capacity* for the strategic ideas it surfaces (a palette of candidate
> features, guards, fixtures) but does **not** pre-commit weights or rewire the
> production evaluator. "Let a thousand flowers bloom" — let weight-learning and
> ablation decide which signals matter.

## Method

BGA blocks scraping, so the move log was transcribed and replayed through our
own `GameState` engine. Two payoffs:

1. **Engine correctness.** All 70 turns (placements, long ring sweeps, marker
   flips, and the full row-completion → marker-removal → ring-removal → handoff
   cycle for both colors) are legal in our engine and the game ends exactly 3–2
   Black. This is now a regression fixture: `yinsh_ml/tests/test_human_game_replay.py`.
2. **Analysis.** Every position is scored with the production 7-feature
   heuristic and ranked against what a greedy heuristic agent would play.

(Incidental finding: BGA prints scores **mover-first** — "mover : opponent" —
not White:Black. Documented in the replay test.)

## Finding 1 — ~41% of the production heuristic is inert in real play

| Feature | Intended weight | Distinct values across the game |
|---|---|---|
| `completed_runs_differential` (#1) | 0.239 | **1 (constant 0)** |
| `potential_runs_count` (#4) | 0.171 | **was 1 (constant 0); now 9 after fix** |

- `completed_runs_differential` reads ~0 at every observable position because a
  completed row is **atomically removed within the same turn** — a standing
  5-row is never seen between moves. Legitimately inert; not a bug. A live
  replacement (e.g. *immediately-completable* rows) is more useful — see
  `near_completion_threats` in the palette.
- `potential_runs_count` was a **latent bug**: it filtered `find_marker_rows()`
  (length ≥ 5 only) for runs of length 3–4 — an unsatisfiable condition, so the
  feature was identically 0 *everywhere*, in every codepath. If the original
  100K-game weight fit ran the same code, the learned weight 0.171 was fit to a
  constant and is spurious. **Fixed** (`yinsh_ml/heuristics/features.py`,
  `_maximal_marker_runs`); now varies −3…+7 over the game. The weight should be
  re-fit; the value was left untouched deliberately.

A reusable guard now catches this class of bug:
`yinsh_ml/heuristics/feature_diagnostics.py::feature_liveness_report`, wired to
the human game in `yinsh_ml/tests/test_feature_liveness.py`.

## Finding 2 — the static eval is anti-correlated with the result

The heuristic rates Black at −40…−80 for ~80% of a game Black *won*
(`docs/game_reviews/bga_862307561_eval_curve.png`). The greedy depth-1 ranking
agrees from the other side: **the agent picked the human's move on only ~13–17%
of Black's decisions.** And the pattern of *when* it agrees is the insight:

- Opening/middlegame (turns 12–36): human moves rank **20th–50th of ~60** — near
  the bottom of the agent's list. The agent wants to shuffle one ring (`D7-…`)
  to greedily max a positional score.
- Sharp endgame (turns 58–70): the agent **agrees** — the winning sequence is
  one it finally "sees" once the position is concrete and tactical.

**Our heuristic is a passable tactician and a poor strategist.** It prices the
1-ply threat; it cannot price a 5–10 move plan. Black won on exactly the plans
it is blind to.

## Why it was a good game (the strategy our stack can't see)

- **Opening — spread vs. cluster.** White put four of five rings on the B-file
  (B4–B7) + F6; Black spread to edges and corners (A5, B1, B3, C6, **K7**).
  White made one concentrated bet (which paid once, turn 31); Black bought
  board-wide optionality.
- **Middlegame — tempo over score.** Black's board-crossing sweeps
  (`B1→K10→E10→E1→E8`, `B3→J11`, `K7→F7`) kept every ring live and threatening
  multiple future rows at once. YINSH's core truth: rings degrade as the board
  fills, so **optionality is the real currency** — and our differential,
  geometry-only features don't price it.
- **Endgame — the sequencing race.** Once both sides started cashing rows, it
  became a flip-war (Black's sweeps escalate ↷4, ↷5, ↷3). Black converted three
  rows in a tighter window than White converted two. The winning `H10-F10`
  completed a `D6-H10` diagonal seeded ~6 moves earlier.

## What was folded in (capacity, not commitment)

| Deliverable | Location |
|---|---|
| Human game as engine regression fixture | `yinsh_ml/data/human_games/bga_862307561.py`, `tests/test_human_game_replay.py` |
| `potential_runs_count` bug fix + tests | `yinsh_ml/heuristics/features.py`, `tests/test_potential_runs_fix.py` |
| Dead-feature guard (liveness report) | `yinsh_ml/heuristics/feature_diagnostics.py`, `tests/test_feature_liveness.py` |
| Experimental feature palette (un-weighted) | `yinsh_ml/heuristics/experimental_features.py`, `tests/test_experimental_features.py` |
| Reproducible review tool | `scripts/review_human_game.py` |

The experimental palette captures the ideas above as distinct, opt-in signals
(not in any default weight set):

- `ring_mobility_differential` — optionality / mobility preservation.
- `ring_confinement_pressure` — ring degradation / death (tail risk).
- `near_completion_threats` — immediately-completable rows (a live stand-in for
  the inert `completed_runs_differential`).
- `defensive_disruption` — the missing **defensive** term: opponent near-rows I
  can sever minus mine they can sever.
- `marker_tempo_differential` — board presence / tempo.

## Abstract directions (the open questions, not answers)

These are recorded as directions to explore, deliberately without picking a
winner:

1. **Don't trust the static heuristic as a value target.** This game is concrete
   evidence it can be confidently wrong-signed for most of a won game. It argues
   for learned value from game outcomes (AlphaZero-style), with the heuristic
   demoted to a policy prior / move-ordering aid. Aligns with
   `TRAINING_REFACTOR_PLAN.md`.
2. **Use strong human games as calibration anchors / curriculum**, and as
   liveness fixtures (a feature constant across a real game is a red flag).
3. **Watch for the agent's single-ring fixation** as a self-play pathology — log
   ring-usage entropy per game as a health metric.
4. **Audit the tactical detectors' false-positive rate.** The `2000`
   "forced ring removal" score fires on moves the human declined and still won;
   it may overvalue answerable threats before we lean on it harder in MCTS.
